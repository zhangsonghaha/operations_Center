# 修复 404 问题的实施方案

服务器管理页面出现 404 的主要原因是**路由结构配置不符合 RuoYi 框架规范**。在 RuoYi 框架中，顶层菜单如果是一个具体的页面（而非目录），必须通过特殊的配置或者包裹在 `Layout` 组件中才能正确显示侧边栏和顶部导航。

我将调整数据库中的菜单结构，采用标准的 **目录 (Directory) -> 菜单 (Menu)** 结构，确保页面能正确嵌入系统布局中。

## 实施步骤

1.  **清理旧数据**：删除之前创建的 "服务器管理" 菜单记录。
2.  **创建父级目录**：
    *   名称：**运维管理**
    *   组件：`Layout` (核心框架布局)
    *   路径：`ops`
    *   类型：`M` (目录)
3.  **创建子级菜单**：
    *   名称：**服务器管理**
    *   组件：`ops/server/index` (对应前端文件)
    *   路径：`server`
    *   类型：`C` (菜单)
    *   权限：`ops:server:list`
4.  **执行修复脚本**：编写并运行 Python 脚本完成上述数据库操作。
5.  **重启提示**：操作完成后，您需要**重启后端服务**并**刷新浏览器**以重新加载路由。

这种结构不仅能解决 404 问题，也为后续扩展其他运维功能（如数据库管理、日志管理）预留了分组空间。
