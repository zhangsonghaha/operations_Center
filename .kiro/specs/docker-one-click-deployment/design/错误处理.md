# 错误处理

### 错误场景 1: Docker 未安装

**条件**: 远程服务器未安装 Docker
**响应**: 
- 环境检测阶段返回 `dockerInstalled = false`
- 前端显示友好提示："目标服务器未安装 Docker，请先安装 Docker Engine"
- 提供 Docker 安装文档链接
**恢复**: 用户需手动安装 Docker 后重试

### 错误场景 2: Docker 服务未运行

**条件**: Docker 已安装但服务未启动
**响应**:
- 环境检测阶段返回 `dockerRunning = false`
- 前端显示提示："Docker 服务未运行"
- 提供启动命令建议：`sudo systemctl start docker`
**恢复**: 自动尝试启动 Docker 服务（需要 sudo 权限）或提示用户手动启动

### 错误场景 3: 容器名冲突

**条件**: 目标服务器已存在同名容器
**响应**:
- 部署前检查阶段抛出异常
- 推送日志："容器名称已存在: {containerName}"
- 前端显示错误并提供选项：
  - 使用不同的容器名
  - 删除现有容器后重新部署
**恢复**: 用户选择操作后重试

### 错误场景 4: 端口冲突

**条件**: 主机端口已被其他进程占用
**响应**:
- 端口检查阶段抛出异常
- 推送日志："端口已被占用: {port}"
- 查询占用端口的进程信息并显示
**恢复**: 
- 用户修改端口映射配置
- 或停止占用端口的进程

### 错误场景 5: 镜像拉取失败

**条件**: 网络问题、镜像不存在、认证失败
**响应**:
- 捕获拉取异常
- 推送详细错误日志
- 区分错误类型：
  - 网络超时：建议检查网络连接或使用镜像加速器
  - 404错误：镜像名或标签不存在
  - 401错误：需要认证，提示配置私有仓库凭证
**恢复**: 
- 自动重试（最多3次）
- 失败后回滚，清理部分下载的镜像层

### 错误场景 6: 容器启动失败

**条件**: 配置错误、资源不足、依赖缺失
**响应**:
- 捕获容器创建异常
- 获取容器日志并推送
- 推送日志："容器启动失败，查看详细日志"
**恢复**:
- 自动删除失败的容器
- 保留部署日志供排查
- 建议用户检查配置和资源

### 错误场景 7: 健康检查超时

**条件**: 容器启动但健康检查未通过
**响应**:
- 推送警告日志："健康检查未通过，但容器已启动"
- 不回滚容器（容器可能需要更长时间初始化）
- 标记部署状态为 "WARNING"
**恢复**: 
- 用户可手动检查容器日志
- 可选择停止或继续观察

### 错误场景 8: SSH 连接失败

**条件**: 网络问题、认证失败、服务器不可达
**响应**:
- 在获取 SSH 会话阶段抛出异常
- 推送日志："SSH连接失败: {errorMessage}"
- 前端显示连接错误详情
**恢复**:
- 自动重试连接（最多3次）
- 提示用户检查服务器配置和网络

### 错误场景 9: 磁盘空间不足

**条件**: 服务器磁盘空间不足以拉取镜像
**响应**:
- 环境检测阶段检查可用空间
- 如果可用空间 < 5GB，显示警告
- 如果拉取过程中空间不足，捕获异常
**恢复**:
- 提示用户清理磁盘空间
- 提供 Docker 清理命令：`docker system prune -a`

### 错误场景 10: 权限不足

**条件**: SSH 用户无权限执行 Docker 命令
**响应**:
- 捕获权限相关错误（"permission denied"）
- 推送日志："权限不足，无法执行Docker命令"
- 提示将用户添加到 docker 组：`sudo usermod -aG docker {username}`
**恢复**: 用户配置权限后重试
