# 测试策略

### 单元测试

**测试范围**:
- 所有 Service 层方法
- Docker 命令构建逻辑
- 配置验证逻辑
- 数据转换和解析方法

**关键测试用例**:

1. **buildDockerRunCommand 测试**
   - 测试基本配置生成正确命令
   - 测试端口映射格式
   - 测试环境变量转义
   - 测试卷挂载路径处理
   - 测试特殊字符转义

2. **checkDockerEnvironment 测试**
   - Mock SSH 会话
   - 测试 Docker 已安装场景
   - 测试 Docker 未安装场景
   - 测试 Docker 未运行场景
   - 测试命令执行异常场景

3. **deployContainer 测试**
   - Mock 完整部署流程
   - 测试成功部署场景
   - 测试各种失败场景
   - 测试回滚逻辑
   - 验证日志推送调用

4. **端口冲突检查测试**
   - 测试端口已占用场景
   - 测试端口可用场景
   - 测试命令不可用场景

### 集成测试

**测试范围**:
- Controller 到 Service 的完整调用链
- 数据库操作
- Redis 缓存
- WebSocket 日志推送

**测试环境**:
- 使用 TestContainers 启动 MySQL 和 Redis
- Mock SSH 连接和 Docker 命令执行
- 使用 @SpringBootTest 加载完整上下文

**关键测试场景**:

1. **完整部署流程测试**
   - 创建部署请求
   - 验证环境检测
   - 验证镜像拉取
   - 验证容器创建
   - 验证数据库记录
   - 验证日志推送

2. **容器管理测试**
   - 测试启动/停止/重启操作
   - 测试容器删除
   - 测试日志获取
   - 测试状态查询

3. **模板功能测试**
   - 测试模板创建
   - 测试从模板生成配置
   - 测试模板参数替换

### 端到端测试

**测试范围**:
- 前端到后端的完整用户流程
- 真实 Docker 环境交互（使用测试服务器）

**测试场景**:

1. **用户部署 MySQL 容器**
   - 登录系统
   - 选择服务器
   - 选择 MySQL 模板
   - 配置参数
   - 执行部署
   - 查看实时日志
   - 验证容器运行
   - 验证端口可访问

2. **容器监控和管理**
   - 查看容器列表
   - 查看容器状态
   - 查看容器日志
   - 停止容器
   - 启动容器
   - 删除容器

### 性能测试

**测试指标**:
- 部署响应时间（目标：< 60秒，不含镜像拉取）
- 并发部署支持（目标：支持10个并发部署）
- 日志推送延迟（目标：< 500ms）
- 容器列表查询性能（目标：< 2秒）

**测试方法**:
- 使用 JMeter 进行压力测试
- 模拟多用户并发部署
- 监控系统资源使用
- 分析性能瓶颈

### 安全测试

**测试范围**:
- 命令注入防护
- SQL 注入防护
- XSS 防护
- 权限验证

**测试用例**:

1. **命令注入测试**
   - 在容器名中注入恶意命令
   - 在环境变量中注入命令
   - 在卷路径中注入命令
   - 验证所有输入都被正确转义

2. **权限测试**
   - 测试未授权用户访问
   - 测试跨租户数据访问
   - 测试敏感操作权限控制
