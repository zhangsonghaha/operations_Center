# 性能考虑

### 性能优化策略

**1. SSH 连接池**
- 复用现有的 SSH 连接，避免频繁建立连接
- 实现连接池管理，支持并发操作
- 设置合理的连接超时和保活机制

**2. 镜像缓存**
- 在数据库中缓存已拉取的镜像信息
- 部署前检查镜像是否已存在，避免重复拉取
- 定期清理未使用的镜像缓存记录

**3. 异步部署**
- 部署操作异步执行，立即返回部署任务ID
- 通过 WebSocket 实时推送部署进度
- 支持查询部署任务状态

**4. 批量操作优化**
- 支持批量查询容器状态（单次 SSH 调用）
- 使用 `docker ps --format json` 减少解析开销
- 批量操作使用事务保证一致性

**5. 日志推送优化**
- 使用 Redis 作为日志消息队列
- 批量推送日志，减少 WebSocket 消息数量
- 限制日志推送频率（最多每秒10条）

**6. 数据库查询优化**
- 为常用查询字段添加索引
- 使用 Redis 缓存热点数据（模板、服务器信息）
- 分页查询容器列表，避免一次加载过多数据

### 性能指标

| 操作 | 目标性能 | 说明 |
|------|---------|------|
| 环境检测 | < 5秒 | 包含 Docker 版本、状态、磁盘空间检查 |
| 镜像拉取 | 取决于网络 | 小镜像(< 100MB) 目标 < 2分钟 |
| 容器创建 | < 10秒 | 不含镜像拉取时间 |
| 健康检查 | < 30秒 | 可配置超时时间 |
| 容器列表查询 | < 2秒 | 单服务器，< 100个容器 |
| 容器状态查询 | < 1秒 | 单个容器 |
| 日志推送延迟 | < 500ms | 从生成到前端接收 |
| 并发部署 | 10+ | 支持同时部署的容器数量 |

### 资源限制

**后端资源**:
- 每个部署任务占用1个线程
- SSH 连接池最大连接数：50
- WebSocket 最大连接数：200
- Redis 日志队列最大长度：10000

**前端资源**:
- 日志查看器最大显示行数：1000（超出自动滚动）
- 容器列表分页大小：20
- 实时监控刷新间隔：5秒
