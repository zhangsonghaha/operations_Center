# 安全考虑

### 安全威胁与防护

**威胁 1: 命令注入攻击**

**风险**: 恶意用户通过容器名、环境变量等注入 Shell 命令

**防护措施**:
- 所有用户输入必须经过严格验证和转义
- 使用白名单验证容器名（只允许字母、数字、下划线、连字符）
- 环境变量值使用单引号包裹并转义特殊字符
- 路径参数使用绝对路径并验证格式
- 禁止在命令中使用管道、重定向等特殊字符

**实现**:
```java
private String escapeShellArg(String arg) {
    if (arg == null) return "";
    // 移除危险字符
    arg = arg.replaceAll("[;&|`$(){}\\[\\]<>]", "");
    // 转义单引号
    arg = arg.replace("'", "'\\''");
    // 用单引号包裹
    return "'" + arg + "'";
}

private boolean isValidContainerName(String name) {
    return name != null && name.matches("^[a-zA-Z0-9_-]+$");
}
```

**威胁 2: 权限提升**

**风险**: 用户通过容器配置获取主机 root 权限

**防护措施**:
- 禁止挂载敏感主机路径（/etc, /root, /var/run/docker.sock）
- 限制容器特权模式（--privileged）的使用
- 强制使用非 root 用户运行容器（--user）
- 限制容器能力（--cap-drop ALL --cap-add 必要能力）

**实现**:
```java
private void validateVolumeMounts(List<VolumeMount> volumes) {
    List<String> forbiddenPaths = Arrays.asList(
        "/etc", "/root", "/var/run/docker.sock", "/sys", "/proc"
    );
    
    for (VolumeMount volume : volumes) {
        for (String forbidden : forbiddenPaths) {
            if (volume.getHostPath().startsWith(forbidden)) {
                throw new SecurityException(
                    "禁止挂载敏感路径: " + volume.getHostPath()
                );
            }
        }
    }
}
```

**威胁 3: 资源耗尽攻击**

**风险**: 恶意用户创建大量容器或不限制资源的容器

**防护措施**:
- 强制设置容器资源限制（CPU、内存）
- 限制单用户可创建的容器数量
- 限制单服务器可部署的容器数量
- 监控资源使用，超限自动停止容器

**实现**:
```java
private void enforceResourceLimits(DockerDeployConfig config) {
    if (config.getResources() == null) {
        config.setResources(new ResourceLimit());
    }
    
    ResourceLimit resources = config.getResources();
    
    // 强制设置默认限制
    if (resources.getCpuLimit() == null) {
        resources.setCpuLimit("2.0");  // 最多2核
    }
    if (resources.getMemoryLimit() == null) {
        resources.setMemoryLimit("2g");  // 最多2GB
    }
    
    // 验证限制不超过最大值
    double cpuLimit = Double.parseDouble(resources.getCpuLimit());
    if (cpuLimit > 4.0) {
        throw new SecurityException("CPU限制不能超过4核");
    }
}
```

**威胁 4: 敏感信息泄露**

**风险**: 容器配置、环境变量、日志中包含敏感信息

**防护措施**:
- 密码、密钥等敏感信息加密存储
- 日志中脱敏敏感信息（密码、Token）
- 限制日志查看权限
- 使用 Docker Secrets 管理敏感配置

**实现**:
```java
private String maskSensitiveInfo(String log) {
    // 脱敏密码
    log = log.replaceAll(
        "(?i)(password|passwd|pwd)=\\S+", 
        "$1=******"
    );
    // 脱敏 Token
    log = log.replaceAll(
        "(?i)(token|secret|key)=\\S+", 
        "$1=******"
    );
    return log;
}
```

**威胁 5: 未授权访问**

**风险**: 用户访问或操作不属于自己的容器

**防护措施**:
- 基于 RBAC 的权限控制
- 容器操作前验证用户权限
- 记录所有操作审计日志
- 支持多租户隔离

**实现**:
```java
@PreAuthorize("@ss.hasPermi('docker:container:deploy')")
public DockerDeployResult deployContainer(DockerDeployConfig config) {
    // 验证用户是否有权限访问目标服务器
    if (!hasServerPermission(getCurrentUserId(), config.getServerId())) {
        throw new SecurityException("无权限访问该服务器");
    }
    // ... 部署逻辑
}
```
