# 正确性属性

*属性是一个特征或行为，应该在系统的所有有效执行中保持为真——本质上是关于系统应该做什么的正式陈述。属性作为人类可读规范和机器可验证正确性保证之间的桥梁。*

### 属性 1: 部署幂等性

*For any* 有效的部署配置，如果使用该配置成功部署一个容器，那么使用相同配置再次部署应该失败并提示容器名已存在。

**验证需求: 需求 2.3, 需求 4.9**

### 属性 2: 容器名唯一性

*For any* 服务器和该服务器上的任意两个容器，如果两个容器的名称相同，那么它们必须是同一个容器（具有相同的容器 ID）。

**验证需求: 需求 2.2, 需求 2.3**

### 属性 3: 端口映射一致性

*For any* 状态为 running 的容器，该容器配置中的所有主机端口都应该在目标服务器上处于被占用状态。

**验证需求: 需求 4.3, 需求 8.3**

### 属性 4: 镜像存在性

*For any* 状态为 running 的容器，该容器使用的镜像（镜像名和标签）必须存在于目标服务器上。

**验证需求: 需求 3.1, 需求 4.1**

### 属性 5: 部署日志完整性

*For any* 状态为 SUCCESS 的部署，其关联的部署日志必须包含所有关键步骤的日志消息：Docker环境检测通过、镜像拉取完成、容器创建成功和部署完成。

**验证需求: 需求 6.3, 需求 6.8**

### 属性 6: 资源清理保证

*For any* 状态为 FAILED 且已创建容器的部署，该容器要么已被删除（不存在于服务器上），要么部署日志中包含"容器清理失败"的消息。

**验证需求: 需求 11.2, 需求 11.3, 需求 11.4**

### 属性 7: 健康检查超时

*For any* 健康检查，如果当前时间超过健康检查开始时间加上超时时间，那么健康检查结果必须为失败。

**验证需求: 需求 5.2, 需求 5.9**

### 属性 8: 命令注入防护

*For any* 部署配置，构建的 Docker 命令中不应包含命令连接符（&&、||、;、|），并且所有用户输入的值都应该被正确转义。

**验证需求: 需求 10.1, 需求 10.5, 需求 10.6**

### 属性 9: 环境检测完整性

*For any* 环境检测请求，如果 Docker 已安装且服务运行正常，那么返回的环境信息必须包含版本、存储驱动、根目录、总空间和已用空间。

**验证需求: 需求 1.6, 需求 1.7, 需求 1.8**

### 属性 10: 端口冲突检测

*For any* 部署配置，如果配置中的任何主机端口已被占用，那么部署必须在端口检查阶段失败。

**验证需求: 需求 2.4, 需求 2.5**

### 属性 11: 敏感信息脱敏

*For any* 部署日志或容器日志，所有包含密码、密钥或 Token 的内容都应该被替换为 `******`。

**验证需求: 需求 6.6, 需求 6.7, 需求 8.9, 需求 10.7**

### 属性 12: 权限验证

*For any* 容器操作请求，只有具有相应权限的用户才能成功执行操作，否则应该返回权限错误。

**验证需求: 需求 7.6, 需求 7.7, 需求 10.8**

### 属性 13: SSH 连接清理

*For any* 部署操作，无论成功或失败，SSH 连接都必须在操作结束时被关闭。

**验证需求: 需求 11.8, 需求 12.1**

### 属性 14: 镜像拉取幂等性

*For any* 镜像拉取请求，如果目标镜像已存在于服务器上，那么系统应该跳过拉取并推送相应的日志消息。

**验证需求: 需求 3.1, 需求 3.2**

### 属性 15: 容器状态一致性

*For any* 容器，数据库中记录的容器状态应该与 Docker Engine 中该容器的实际状态保持一致。

**验证需求: 需求 7.8, 需求 15.6**
