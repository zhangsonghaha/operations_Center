# 与现有系统集成

### 集成点 1: 服务器管理

**现有功能**: `t_ops_server` 表和 `OpsServer` 实体

**集成方式**:
- 直接使用现有的服务器配置
- 复用 SSH 连接逻辑
- 在服务器列表中标识是否支持 Docker

**修改**:
```sql
-- 为服务器表添加 Docker 支持标识（可选）
ALTER TABLE t_ops_server 
ADD COLUMN docker_enabled CHAR(1) DEFAULT '0' COMMENT 'Docker支持(0否 1是)';
```

### 集成点 2: 部署日志系统

**现有功能**: `t_ops_deploy_log` 表和 WebSocket 实时日志

**集成方式**:
- 复用现有的部署日志表结构
- 使用相同的 WebSocket 推送机制
- 添加部署类型字段区分传统部署和 Docker 部署

**修改**:
```sql
-- 为部署日志表添加部署类型（如果不存在）
ALTER TABLE t_ops_deploy_log 
ADD COLUMN deploy_type VARCHAR(20) DEFAULT 'TRADITIONAL' 
COMMENT '部署类型(TRADITIONAL/DOCKER)';
```

### 集成点 3: 权限系统

**现有功能**: Spring Security + RBAC

**集成方式**:
- 添加 Docker 相关权限定义
- 在菜单表中添加 Docker 管理菜单
- 使用 `@PreAuthorize` 注解控制接口权限

**权限定义**:
```sql
-- 插入 Docker 管理菜单
INSERT INTO sys_menu VALUES (
    2100, 'Docker管理', 0, 6, 'docker', NULL, NULL, 1, 0, 'M', '0', '0', 
    '', 'docker', 'admin', sysdate(), '', NULL, 'Docker容器管理'
);

-- 插入子菜单和权限
INSERT INTO sys_menu VALUES (
    2101, '容器部署', 2100, 1, 'deploy', 'docker/deploy/index', NULL, 1, 0, 
    'C', '0', '0', 'docker:deploy:view', 'deploy', 'admin', sysdate(), '', NULL, ''
);

-- 添加操作权限
INSERT INTO sys_menu VALUES (
    2102, '部署容器', 2101, 1, '', NULL, NULL, 1, 0, 'F', '0', '0', 
    'docker:container:deploy', '#', 'admin', sysdate(), '', NULL, ''
);
```

### 集成点 4: 应用管理

**现有功能**: `t_ops_app` 表

**集成方式**:
- 扩展应用类型，支持 Docker 应用
- Docker 容器可关联到应用
- 统一管理传统应用和容器化应用

**修改**:
```sql
-- 更新应用类型字典
INSERT INTO sys_dict_data VALUES (
    NULL, 3, 'Docker容器', 'DOCKER', 'ops_app_type', NULL, 'primary', 'N', 
    '0', 'admin', sysdate(), '', NULL, 'Docker容器化应用'
);
```

### 集成点 5: 监控告警

**现有功能**: 系统监控模块

**集成方式**:
- 容器状态监控集成到现有监控页面
- 容器异常触发告警
- 统一的监控数据展示
